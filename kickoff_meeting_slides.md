# 2. 企画コンセプト

> **「文系も参加しやすく、理系も物足りなさを感じないAIハッカソン」**

単なる技術コンテストではありません。
学生が自ら<b>社会課題・地域課題</b>をリサーチし、<b>AI・ノーコード技術</b>を活用して「社会実装可能なアイデア」へと落とし込む<b>実践型イベント（アントレプレナーシップ教育）</b>です。

<!--
【スピーカーノート】
私たちの企画コンセプトです。
単に「AIを使って面白いものを作ろう」ではなく、明確に「社会課題の解決」と「社会実装」にフォーカスしています。
参加する学生には、ユーザー目線での課題発見からプロトタイプの実装まで、起業家のようなプロセスを体験してもらいたいと考えています。
-->

---

# 3. なぜ「文系・理系混合」なのか？

AI時代の価値創出には、技術力以上に<b>課題定義力</b>と<b>ビジネス設計力</b>が不可欠。
ノーコードAIを使えば、技術経験の有無に関わらず全員がプロトタイプ構築に貢献できます。

- **共通の役割**: 課題定義、解決策の意思決定
- **文系の役割**: 社会課題のリサーチ、ユーザー理解、企画設計、ビジネスモデル検討、プレゼン構成
- **理系の役割**: AI活用設計、ノーコード実装、データ連携、技術的な実現可能性の検証

<!--
【スピーカーノート】
「なぜ文系・理系を混ぜるのか？」についてです。
今のAI時代、コードが書けることと同じくらい、「誰のどんな課題を解くのか」を定義し、事業性を考える力が求められています。
文系学生にはリサーチやビジネス設計を、理系学生には技術選定や実装をリードしてもらい、ノーコードツールを使って全員で形にする。互いの強みが交わることで、より現実に即したソリューションが生まれると考えています。
-->

# 4. 段階的な開催ロードマップ

80名規模の成功に向けて、3ステップで拡大していきます。

1.  **Phase 1：プレ勉強会 ＆ AIアイデアソン（8月想定）**
    *   **規模**: 30〜40名 (8月前半プレイベント、後半アイデアソン)
    *   **目的**: AI活用の成功体験を提供。優秀チームはPhase 2へ。
2.  **Phase 2：本格ハッカソン（9月想定 / 2日間）**
    *   **規模**: 50〜60名
    *   **目的**: コーディングを伴う本格開発。スポンサー連携開始。
3.  **Phase 3：来年以降の年2回・大型開催（長期スケール化）**
    *   **規模**: 春・夏大型開催
    *   **目的**: 関西最大級のフラグシップ開催へのスケール。

<!--
【スピーカーノート】
いきなり80名を集めるのはリスクが高いので、3段階のロードマップを描いています。
まずは8月前半に勉強会やLT会で集客し、8月後半にアイデアソンを実施。優秀チームにはシード権を与え、9月のPhase 2へと繋げていきます。フェーズ3の大規模化は焦らず来期に持ち越し、今期は2段階で強固な基盤を作ります。

# 7. 【企画・進行班】役割とタスク

# 10. 決定権の境界線（裁量とエスカレーション）

私たちコアメンバーは「学生主体」で運営しますが、トラブルを防ぐため決定権の境界線を明確にします。

### 🟢 自分たちの班で自由に決定して良いこと（裁量権）
- 各タスクの具体的な進め方やスケジュール調整
- デザインのトーン＆マナー、PRの文言、技術ツールの選定
- 当日ボランティアへの指示出し

### 🔴 PM・田窪氏への「事前承認・相談」が必要なこと
- <b>予算</b>が発生する、または予算を超過する事項の決定
- 企画の<b>MUST要件</b>（日程、会場、参加費無料など）を変更すること
- 外部パートナーとの<b>公式な条件交渉・最終決定</b>（※アプローチ自体は外部連携班が進めるが、最終調整はPM等と連携）

<!--
【スピーカーノート】
最後に、一番重要な「決定権の境界線」についてです。
基本的に、各班の専門領域（デザイン、技術、PRの手法など）については、それぞれの裁量で自由に決めて進めてください。皆さんのセンスを信頼しています。
ただし、「お金に関わること」「イベントの前提条件（日程や会場）が変わること」「企業や自治体といった外部への公式な連絡」については、必ず統括PMや田窪さんにエスカレーションして事前承認を取ってください。これを守ることで、余計なトラブルを防ぎつつスピード感を持って進められます。
